# Metric Definition

**GEO Monitor Toolkit** 采用 4 个核心指标：`mention_rate`、`positive_mention_rate`、`capability_accuracy`、`ecosystem_accuracy`。底层标注采用 **0-2** 三档，是因为它比二元判断更能保留“部分命中 / 模糊正确”的中间状态，同时又比 0-5 细分量表更稳定，能降低跨标注人的漂移风险。

## 为什么用 0-2

| 分值 | 含义 | 适用解释 |
|---|---|---|
| 0 | 未命中 / 明显错误 / 缺失 | 模型没有提到，或给出错误结论 |
| 1 | 部分命中 / 有保留地正确 | 提到但不完整，或方向对但细节不足 |
| 2 | 明确命中 / 准确完整 | 提及充分且判断正确 |

## 四个指标如何聚合

在单条 query 级别，标注员先给出 0-2 分；在 run 级别，再将“拿到 2 分的占比”与“拿到 1 或 2 分的宽松占比”分别聚合为百分比。因此仓库里的 `summary.json` 里会同时出现严格指标与宽松指标。

## 与业务 KPI 的映射

| GEO 指标 | 对应业务问题 | 可映射 KPI |
|---|---|---|
| Mention Rate | 模型有没有想到你 | 品牌提及率、入选率 |
| Positive Mention Rate | 模型提到你时是否正向 | 正向声量、推荐率 |
| Capability Accuracy | 模型是否正确理解你的能力边界 | 能力认知准确率、错误成本 |
| Ecosystem Accuracy | 模型是否正确理解你的生态、集成与适用场景 | 生态认知准确率、线索质量 |

## 为什么不是直接用业务 KPI

业务 KPI 往往是最终结果，如注册、激活、Star、试用转化；而 GEO 指标更像是上游认知层信号。它们适合作为 **内容铺设、负向修复、问答覆盖** 的先导指标，帮助团队在流量与转化发生前先看到模型侧认知是否改善。
